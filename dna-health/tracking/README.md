# Tracking System

Use GitHub Issues to run your weekly check-ins and lab tracking.

## How to Create a Weekly Check-In

1. Go to **Issues** → **New Issue** → select **Weekly Check-In** template
2. Fill in the week date in the title: `Weekly Check-In — 2026-06-21`
3. Complete the checklist and symptom scores
4. Add label: `weekly-checkin`
5. Close the issue at end of week

## How to Log New Lab Results

1. Open [lab-results-log.md](lab-results-log.md) directly and edit the tables
2. Or create a GitHub Issue using the **Lab Results** template
3. Add label: `lab-results`

## Labels to Use

| Label | Color | Purpose |
|-------|-------|---------|
| `weekly-checkin` | Blue | Weekly progress tracking |
| `lab-results` | Purple | New blood/hair test results |
| `supplement-change` | Yellow | Starting or stopping a supplement |
| `symptom-alert` | Red | New or worsening symptoms to monitor |
| `priority` | Orange | Items needing immediate action |

## Milestones

Set quarterly milestones to review overall progress:
- Q3 2026: Baseline labs completed
- Q4 2026: All Priority 1 supplements stable; retest Lp(a) and homocysteine
- Q1 2027: First annual thyroid antibody retest
- Q2 2027: Full annual labs
